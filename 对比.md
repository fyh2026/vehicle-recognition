# 项目对比：人脸识别 vs 车辆识别

## 概览

| | 人脸识别 | 车辆识别 |
|---|---|---|
| 代码量 | 5 个文件 / ~600 行 | 1 个文件 / ~280 行 |
| 依赖 | 2 个（opencv, numpy） | 5 个（YOLO + supervision + torch + opencv + numpy） |

## 技术代差

```
人脸识别:  Haar Cascade + LBPH  →  2010 年代传统 CV
车辆识别:  YOLO11 + ByteTrack   →  2024-25 年深度学习
          ↑ 两个时代的技术
```

## 详细对比

| 维度 | 人脸识别（旧） | 车辆识别（新） |
|------|:--:|:--:|
| 技术路线 | Haar 级联 + LBPH 局部二值模式 | YOLO11 单阶段检测 + ByteTrack 多目标跟踪 |
| 上手门槛 | 需要采集 100-200 张人脸 → 训练模型 → 才能用 | 预训练模型，开箱即用，零训练 |
| 精度 | LBPH 受光照/角度影响大，准确率上限低 | 深度学习模型，COCO 预训练，检测精度远高于传统方法 |
| GPU 加速 | ❌ 纯 CPU | ✅ torch 支持 GPU（也可 CPU） |
| 兼容性 | ❌ OpenCV 4.9+ 已移除 `cv2.face`，必须装 `opencv-contrib` | ✅ 全部主流库，无废弃问题 |
| 识别能力 | 仅检测人脸 + 身份匹配 | 检测 + 车型分类（轿车/摩托车/公交车/卡车） |
| 计数功能 | 无 | 虚拟线双向计数（IN/OUT）+ per-class 统计 |
| 数据输出 | 无 | CSV 事件日志 |
| 实时体验 | 帧率尚可但识别慢 | 跳帧+缩放优化，流畅运行 |
| 演示效果 | 静态识别，终端菜单交互 | 实时画面标注 + 计数面板，视觉冲击力强 |

## 总结

车辆识别项目相比人脸识别项目：
- **技术更现代** — 从 2010 年代传统 CV 升级到 2024 深度学习
- **使用更简单** — 不需要采集数据、训练模型，直接跑
- **精度更高** — YOLO 深度学习 vs LBPH 传统特征
- **展示更强** — 实时画面 vs 终端菜单
